# Объектно-ориентированное программирование (ООП)

Проект представляет собой реализацию на Python объектно-ориентированного программирования.


## Функциональность

### Базовые классы

#### Класс Product
- **Атрибуты**: название, описание, цена, количество
- **Приватные атрибуты**: цена (`_price`)
- **Магические методы**:
  - `__str__` - строковое представление: "Название, цена руб. Остаток: количество шт."
  - `__add__` - сложение продуктов (только одинаковых классов)
- **Свойства**:
  - `price` - геттер и сеттер с валидацией
- **Класс-методы**:
  - `new_product()` - создание продукта из словаря
  - `new_product_with_duplicate_check()` - создание с проверкой дубликатов

#### Класс Category
- **Атрибуты**: название, описание, список товаров
- **Приватные атрибуты**: список товаров (`__products`)
- **Атрибуты класса**: 
  - `category_count` - количество категорий
  - `product_count` - количество товаров
- **Магические методы**:
  - `__str__` - строковое представление: "Название категории, количество продуктов: X шт."
  - `__iter__` - итерация по товарам категории
- **Методы**:
  - `add_product()` - добавление товара с проверкой типа
- **Свойства**:
  - `products` - геттер для форматированного вывода товаров
  - `products_list` - геттер для списка объектов товаров

### Классы-наследники

#### Smartphone (наследуется от Product)
- **Дополнительные атрибуты**:
  - `efficiency` - производительность
  - `model` - модель
  - `memory` - объем встроенной памяти
  - `color` - цвет

#### LawnGrass (наследуется от Product)
- **Дополнительные атрибуты**:
  - `country` - страна-производитель
  - `germination_period` - срок прорастания
  - `color` - цвет

### Особенности реализации

- **Приватные атрибуты**: защита данных от прямого доступа
- **Валидация**: проверка цены и типов при добавлении
- **Наследование**: переиспользование кода через наследование
- **Магические методы**: улучшенное взаимодействие с объектами
- **Итератор**: возможность перебора товаров в категории
